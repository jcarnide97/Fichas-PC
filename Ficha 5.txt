Exercicio 1:

Router R1:

enable
config t

interface Ethernet0
ip address 192.168.X.254 255.255.255.0
no shutdown

interface FastEthernet0
ip address 172.16.X.253 255.255.255.0
no shutdown


router ospf 100
network 192.168.X.0 0.0.0.255 area 0

router bgp 3000
no synchronization
bgp dampening
redistribute ospf 100
network 172.16.X.0 mask 255.255.255.0

neighbor 172.16.X.254 remote-as 1000

Pronto, e depois é dar pings dentro do programa mesmo de uns pc's para os outros e ver se dá, em principio
se os valores estiverem certos deve dar bem. Ele a nos pediu para mostrar a ip route e ip bgp e dizer qual a 
principal diferença, e se virem a ip route apenas mostra uma rota para chegar ao destino, enquanto que a 
ip bgp mostra TODAS as rotas possiveis, ou seja se tiverem no R1 para ire para ISP2 podem ir R1->R2->ISP2
ou podem ir para R1->ISP1->ISP2, e ele mostra isso, ele mostra onde estão e depois o next-hop (ou seja,
o proximo salto, que em para cada valor vai ter dois, porque ele pode sempre ir para dois sitios).

Exericio 2:
O ex2 é bue estupido, porque basicamente o que estão a fazer é a mudar a local preferencia de um router para ele
seguir a outra rota, ou seja, para ir de R1 para ISP2 pode ir por dois caminhos (ver exemplo acima), então
basicamente o que fazem é vao a um dos routers e aumentam ou diminuem a prefrencia (a default é 100, por isso
se aumentarem num ele vai mais para esse mas se diminuirem no outro ele vai mais pelo outro).

Pedem para ser em relaçao a ISP1, ou seja, se queremos que vai mais para ISP1 ou aumentamos a preferencia pelo R1
ou diminuimos no R2, nos diminuimos no R2 porque é como está no enunciado.

Então voltamos ao router R2 e fazemos:

router bgp 3000
neighbor 192.168.X.254 route-map PREF in
route-map PREF permit 10
set local-preference 1 // aqui metemos 1 mas pode ser o que vocês quiserem desde que seja menor que 100
